# Essay Scaffolder

Produces structured skeleton answers for essay exam questions. Guides the student on WHAT to write, not writing it for them.

---

## Trigger

- Student provides an essay question (real or practice)
- Student says "scaffold this question", "help me plan this essay", "what should I cover?"
- After `lecture-mastery-tester` identifies essay readiness as "Needs work"

---

## Input

- The essay question (provided by student or generated by practice question system)
- Relevant flashcards (from the lectures the question spans)
- Source materials (transcripts, slides) for verification
- Course-specific flashcard skill for exam format details

---

## Output Structure

For each essay question, produce:

```markdown
## Essay Scaffold: [Question summary]

### Question Analysis
- **Type:** [mechanism / comparison / clinical / evidence-based]
- **Scope:** Lectures [X, Y] — [topics covered]
- **What's being asked:** [Restate in plain language]
- **Grading cues:** [How many examples? How many differences? What evidence?]

### Opening (1-2 sentences)
[Define the core concept. This is your thesis statement.]
- Key term: [term] — [2-sentence definition from glossary if applicable]

### Body — Required Points

**Point 1: [Label]**
- Argument: [one-line summary]
- Evidence: [study or clinical case to cite]
- Key term: [term to use]

**Point 2: [Label]**
- Argument: [one-line summary]
- Evidence: [study or clinical case to cite]
- Key term: [term to use]

**Point 3: [Label]**
- Argument: [one-line summary]
- Evidence: [study or clinical case to cite]
- Key term: [term to use]

### Closing (1-2 sentences)
[Theoretical implication or synthesis. What does this tell us about the broader system?]

### Terms You MUST Use
- [Term 1], [Term 2], [Term 3], ...

### Common Mistakes to Avoid
- [Mistake 1: e.g., confusing X with Y]
- [Mistake 2: e.g., forgetting to give 3 examples when asked]

### Word Budget
- Opening: ~50 words
- Point 1: ~120 words
- Point 2: ~120 words
- Point 3: ~120 words
- Closing: ~40 words
- **Total: ~450 words** (within 400-500 target)
```

---

## Scaffold Calibration by Course

### AD Clinical Neuropsychology
- Target: 400-500 words
- Always include 3 examples/points if the question asks for them (maps to 6-point rubric)
- Reference glossary terms — the grader expects precise terminology
- Include clinical cases where relevant

### AB Affective Neuroscience
- Scenario-based — the scaffold must address the specific scenario
- Include brain region ↔ function mappings
- Cite empirical studies with (Author, Year)
- Explain mechanisms, not just name them

### Other Courses
- Adapt to the exam format described in the course-specific flashcard skill
- If no essay format exists (e.g., AC CompNeuro uses MCQ), this skill is not applicable

---

## Practice Mode

When no specific question is provided, generate a practice essay question from the lecture material:

1. Select a major topic from the lecture
2. Construct a question in the style of previous exam questions
3. Provide the scaffold
4. After the student writes their answer, evaluate against the scaffold

---

## Evaluation Mode

When the student submits a written essay answer:

```markdown
## Essay Evaluation

### Score: [X/6]

### What You Did Well
- [Specific strength]

### What's Missing
- [Missing point or example]
- [Missing key term]

### How to Improve
- [Actionable suggestion]

### Model Comparison
[Brief comparison to what a full-marks answer would include]
```

Scoring rubric (for AD 6-point scale):
- 6/6: All requested points, correct mechanisms, precise terms, evidence cited
- 5/6: All points present but one lacks depth or evidence
- 4/6: One point missing or significantly weak
- 3/6: Two points missing or major conceptual error
- 2/6: Only one point adequately addressed
- 1/6: Attempt made but fundamentally confused
- 0/6: No relevant content

---

## What NOT to Do

- Never write the full essay for the student — provide structure, not prose
- Never include more points than the question asks for — if it says "3 examples", scaffold exactly 3
- Never suggest content from lectures the question doesn't cover
- Never omit the word budget — students need to know how much to write per section
- Never skip "Common Mistakes" — this is the most actionable section
- Never evaluate harshly without explaining how to improve
